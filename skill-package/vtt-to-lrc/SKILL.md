# VTT to LRC

使用打包在 `scripts/` 里的 Rust 可执行文件完成离线转换。当前包内提供的是 macOS Apple Silicon 二进制。

## 执行步骤

1. 确认用户给出的路径存在，并区分是文件还是目录。
2. 优先运行 `scripts/convert.sh`。
3. 将用户提供的文件或目录路径原样作为参数传给脚本。
4. 如果用户没有提供参数但明确要转换当前目录，就直接无参数运行脚本。
5. 汇总标准输出里的成功文件和标准错误里的失败文件，明确说明输出 `.lrc` 与源 `.vtt` 同目录、同名不同后缀。

## 命令

```bash
./scripts/convert.sh <file-or-dir> [...]
```

当前目录批量扫描：

```bash
./scripts/convert.sh
```

## 转换规则

- 将 `HH:MM:SS.mmm` 转成 `[MM:SS.cc]`，小时折叠进分钟，毫秒截成两位百分秒。
- 去除 `<b>`、`<i>`、`<c>`、`<u>`、`<ruby>`、`<rt>` 及其带属性形式的标签。
- 将同一个 cue 的多行文本拼成一行，中间用空格连接。
- 自动检测并解码 UTF-8（含 BOM）、ASCII、GBK、Windows-1252、Latin1。
- 目录参数使用递归扫描，最大深度为 10；无参数时只扫描当前目录下的 `.vtt` 文件，不递归。
- 单个文件失败时继续处理其他文件；只要存在失败，进程退出码就是 `1`。

## 汇报要求

- 成功时列出生成的 `.lrc` 路径。
- 部分失败时同时给出成功列表和失败原因。
- 如果运行环境不是 macOS arm64，直接说明这个 skill 包当前只内置了该平台的二进制，不能假装成功。
